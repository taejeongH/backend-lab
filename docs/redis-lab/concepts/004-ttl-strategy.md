## TTL

TTL은 캐시 데이터가 Redis에 유지되는 시간을 의미한다.  
TTL을 어떻게 설정하느냐에 따라 데이터 최신성과 캐시 효율 사이의 균형이 달라진다.

### TTL이 너무 짧은 경우

#### 장점

- stale cache가 오래 유지되지 않는다.
- 데이터 변경이 빠르게 반영될 가능성이 높아진다.
- 최신성이 중요한 데이터에 유리하다.

#### 단점

- Cache Miss가 자주 발생한다.
- DB 조회 횟수가 증가한다.
- 캐시를 사용하는 효과가 줄어든다.

### TTL이 너무 긴 경우

#### 장점

- Cache Hit 비율이 높아진다.
- DB 조회 횟수를 줄일 수 있다.
- 변경이 적고 조회가 많은 데이터에 유리하다.

#### 단점

- 데이터가 변경되어도 이전 캐시가 오래 유지될 수 있다.
- stale cache 지속 시간이 길어진다.
- 실시간성이 중요한 데이터에는 적합하지 않다.

### 정리

TTL은 짧을수록 데이터 최신성에는 유리하지만 캐시 효율은 떨어진다.  
반대로 TTL이 길수록 캐시 효율은 좋아지지만 stale cache가 오래 유지될 수 있다.

따라서 TTL은 모든 데이터에 동일하게 적용하기보다, 데이터의 변경 빈도와 최신성 요구사항에 따라 다르게 설정하는 것이 좋다.
